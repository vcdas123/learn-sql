# SQL DML (Data Manipulation Language) – Detailed Notes

## 1. What is DML?

**DML (Data Manipulation Language)** is the subset of SQL used to **insert, modify, and remove data** inside database tables.

Unlike DDL, DML:

- Works on **rows (data)**, not structure
- Directly changes table contents
- Always reports how many rows were affected

Common DML commands:

- `INSERT`
- `UPDATE`
- `DELETE`
- `TRUNCATE` (often grouped with DML in practice)

---

## 2. INSERT Statement

### Purpose

Adds **new rows** to a table. Data can be inserted:

- Manually (explicit values)
- From another table (INSERT … SELECT)

---

## 3. INSERT – Basic Syntax

```sql
INSERT INTO table_name (column1, column2, column3)
VALUES (value1, value2, value3);
```

### Key Rules

- Column list is optional
- Number of values must match number of columns
- Order of values must match column order

---

## 4. INSERT with All Columns

```sql
INSERT INTO customers (id, first_name, country, score)
VALUES (6, 'Anna', 'USA', NULL);
```

---

## 5. INSERT Multiple Rows

```sql
INSERT INTO customers (id, first_name, country, score)
VALUES
(7, 'Sam', NULL, 100),
(8, 'Max', 'USA', 200);
```

### Output

```
2 rows affected
```

---

## 6. INSERT Without Column List

```sql
INSERT INTO customers
VALUES (9, 'Andreas', 'Germany', 300);
```

⚠️ Works **only if**:

- All columns are provided
- Order matches table definition

---

## 7. INSERT Partial Columns (Nullable Columns Only)

```sql
INSERT INTO customers (id, first_name)
VALUES (10, 'Sara');
```

### Important

- Skipped columns must allow `NULL` or have `DEFAULT`
- You **cannot skip NOT NULL columns**

---

## 8. Common INSERT Mistakes

### Wrong Column Order (Logical Error)

```sql
INSERT INTO customers (id, first_name, country)
VALUES (11, 'USA', 'Max');
```

✔ Accepted by SQL
❌ Logically incorrect data

### Wrong Data Type (Rejected)

```sql
INSERT INTO customers (id, first_name)
VALUES ('Max', 12);
```

❌ Rejected by database

---

## 9. INSERT Using SELECT (Insert from Another Table)

### Purpose

Copy data from a **source table** into a **target table**.

### Syntax

```sql
INSERT INTO target_table (col1, col2, col3)
SELECT expr1, expr2, expr3
FROM source_table;
```

---

## 10. Example: Insert Customers into Persons

```sql
INSERT INTO persons (id, person_name, birth_date, phone)
SELECT
    id,
    first_name,
    NULL,
    'UNKNOWN'
FROM customers;
```

### Notes

- Column names don’t need to match
- Data types must match
- Static values are allowed

---

## 11. UPDATE Statement

### Purpose

Modify **existing rows** in a table.

### Basic Syntax

```sql
UPDATE table_name
SET column1 = value1,
    column2 = value2
WHERE condition;
```

⚠️ **Never omit WHERE** unless you intend to update all rows.

---

## 12. UPDATE Single Row

```sql
UPDATE customers
SET score = 0
WHERE id = 6;
```

Output:

```
1 row affected
```

---

## 13. UPDATE Multiple Columns

```sql
UPDATE customers
SET score = 0,
    country = 'UK'
WHERE id = 10;
```

---

## 14. UPDATE Multiple Rows

```sql
UPDATE customers
SET score = 0
WHERE score IS NULL;
```

### Best Practice (Safety Check)

```sql
SELECT * FROM customers
WHERE score IS NULL;
```

---

## 15. DELETE Statement

### Purpose

Remove **specific rows** from a table.

### Syntax

```sql
DELETE FROM table_name
WHERE condition;
```

⚠️ Omitting `WHERE` deletes **all rows**.

---

## 16. DELETE Example

```sql
DELETE FROM customers
WHERE id > 5;
```

Output:

```
5 rows affected
```

---

## 17. DELETE vs TRUNCATE

### DELETE (Row-by-row)

```sql
DELETE FROM persons;
```

- Slower
- Logged
- Can be rolled back (DB dependent)

### TRUNCATE (Fast reset)

```sql
TRUNCATE TABLE persons;
```

- Very fast
- No row count returned
- Cannot be rolled back
- Resets identity counters

---

## 18. DML Safety Rules

- Always test conditions using `SELECT`
- Check affected row count
- Use transactions when possible
- Avoid mass updates/deletes without filters

---

## 19. DML vs DDL

| DML                    | DDL                   |
| ---------------------- | --------------------- |
| Manipulates data       | Manipulates structure |
| INSERT, UPDATE, DELETE | CREATE, ALTER, DROP   |
| Affects rows           | Affects schema        |

---

## 20. Summary

DML commands allow you to:

- Insert new data
- Modify existing data
- Remove unwanted data

Mastering DML is essential for **real-world SQL usage**, reporting, data cleaning, and backend development.
