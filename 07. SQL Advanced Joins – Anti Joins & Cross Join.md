# SQL Advanced Joins – Anti Joins & Cross Join

This chapter covers **advanced SQL JOIN patterns** used mainly for **existence / non-existence checks** and **Cartesian combinations**. These joins are commonly used in real-world data validation, audits, and analysis.

---

## 0. Tables Used in This Chapter

### customers

```sql
CREATE TABLE customers (
    id INT PRIMARY KEY,
    first_name VARCHAR(50),
    country VARCHAR(50),
    score INT
);
```

### orders

```sql
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    sales INT
);
```

---

## 1. LEFT ANTI JOIN

### Definition

A **LEFT ANTI JOIN** returns **only rows from the left table that do NOT have a match in the right table**.

- Left table = data source
- Right table = lookup only
- Matching rows are **excluded**

> SQL does NOT have a direct `LEFT ANTI JOIN` keyword.
> We simulate it using `LEFT JOIN + WHERE right_key IS NULL`.

---

### Syntax Pattern

```sql
SELECT columns
FROM tableA A
LEFT JOIN tableB B
ON A.key = B.key
WHERE B.key IS NULL;
```

---

### Example Task

**Get all customers who have NOT placed any order**

```sql
SELECT c.id, c.first_name
FROM customers c
LEFT JOIN orders o
ON c.id = o.customer_id
WHERE o.customer_id IS NULL;
```

### Output

| id  | first_name |
| --- | ---------- |
| 4   | Martin     |
| 5   | Peter      |

---

### When to Use

- Find inactive customers
- Detect missing related records
- Data quality checks

---

## 2. RIGHT ANTI JOIN

### Definition

A **RIGHT ANTI JOIN** returns **only rows from the right table that do NOT have a match in the left table**.

- Right table = data source
- Left table = lookup only

> No direct `RIGHT ANTI JOIN` keyword exists.
> We use `RIGHT JOIN + WHERE left_key IS NULL`.

---

### Syntax Pattern

```sql
SELECT columns
FROM tableA A
RIGHT JOIN tableB B
ON A.key = B.key
WHERE A.key IS NULL;
```

---

### Example Task

**Get all orders without matching customers**

```sql
SELECT o.order_id, o.customer_id, o.sales
FROM customers c
RIGHT JOIN orders o
ON c.id = o.customer_id
WHERE c.id IS NULL;
```

### Output

| order_id | customer_id | sales |
| -------- | ----------- | ----- |
| 104      | 6           | 300   |

---

### Recommended Alternative (Preferred)

Use **LEFT JOIN by switching tables**:

```sql
SELECT o.order_id, o.customer_id, o.sales
FROM orders o
LEFT JOIN customers c
ON o.customer_id = c.id
WHERE c.id IS NULL;
```

---

## 3. FULL ANTI JOIN

### Definition

A **FULL ANTI JOIN** returns **rows that do NOT match in either table**.

- Rows in A but not in B
- Rows in B but not in A

This is the **opposite of INNER JOIN**.

---

### Syntax Pattern

```sql
SELECT columns
FROM tableA A
FULL JOIN tableB B
ON A.key = B.key
WHERE A.key IS NULL
   OR B.key IS NULL;
```

---

### Example Task

**Find customers without orders AND orders without customers**

```sql
SELECT c.id, c.first_name, o.order_id, o.customer_id
FROM customers c
FULL JOIN orders o
ON c.id = o.customer_id
WHERE c.id IS NULL
   OR o.customer_id IS NULL;
```

### Output

| id   | first_name | order_id | customer_id |
| ---- | ---------- | -------- | ----------- |
| 4    | Martin     | NULL     | NULL        |
| 5    | Peter      | NULL     | NULL        |
| NULL | NULL       | 104      | 6           |

---

## 4. Advanced INNER JOIN (Without Using INNER JOIN)

### Goal

Return **only matching rows** without using `INNER JOIN`.

---

### Solution Using LEFT JOIN

```sql
SELECT c.id, c.first_name, o.order_id, o.sales
FROM customers c
LEFT JOIN orders o
ON c.id = o.customer_id
WHERE o.customer_id IS NOT NULL;
```

### Output

| id  | first_name | order_id | sales |
| --- | ---------- | -------- | ----- |
| 1   | Maria      | 101      | 1200  |
| 2   | John       | 102      | 800   |
| 3   | George     | 103      | 600   |

---

## 5. CROSS JOIN

### Definition

A **CROSS JOIN** returns **all possible combinations** of rows from both tables.

- No matching condition
- Result size = rows(A) × rows(B)

---

### Syntax

```sql
SELECT columns
FROM tableA
CROSS JOIN tableB;
```

---

### Example Task

**Generate all possible combinations of customers and orders**

```sql
SELECT c.first_name, o.order_id
FROM customers c
CROSS JOIN orders o;
```

### Output Size

- Customers = 5 rows
- Orders = 4 rows
- Result = **20 rows**

### Sample Output

| first_name | order_id |
| ---------- | -------- |
| Maria      | 101      |
| Maria      | 102      |
| Maria      | 103      |
| Maria      | 104      |
| ...        | ...      |

---

## 6. Summary

| Join Type  | Purpose                    |
| ---------- | -------------------------- |
| LEFT ANTI  | Find rows missing on right |
| RIGHT ANTI | Find rows missing on left  |
| FULL ANTI  | Find all unmatched rows    |
| CROSS JOIN | Generate combinations      |

---

## Key Takeaways

- Anti joins = `JOIN + WHERE IS NULL`
- LEFT JOIN is preferred over RIGHT JOIN
- CROSS JOIN can create massive results
- Advanced joins are mostly used for **data validation and audits**
