# Window Ranking Functions in SQL

---

# 1. Introduction

Window ranking functions assign a rank to each row within a window.

They are divided into **two main categories**:

1. Integer-Based Ranking
2. Percentage-Based Ranking

SQL always **sorts the data first** before assigning ranks.

# 3. Ranking Functions Overview

## 3.1 Integer-Based Ranking

| Function     | Handles Ties         | Leaves Gaps | Unique Rank |
| ------------ | -------------------- | ----------- | ----------- |
| ROW_NUMBER() | ❌ No                | ❌ No       | ✅ Yes      |
| RANK()       | ✅ Yes               | ✅ Yes      | ❌ No       |
| DENSE_RANK() | ✅ Yes               | ❌ No       | ❌ No       |
| NTILE(n)     | Divides into buckets | N/A         | N/A         |

## 3.2 Percentage-Based Ranking

| Function       | Formula Type                | Inclusive / Exclusive |
| -------------- | --------------------------- | --------------------- |
| CUME_DIST()    | position / total_rows       | Inclusive             |
| PERCENT_RANK() | (position-1)/(total_rows-1) | Exclusive             |

---

# 4. Syntax Rules (Transcript Accurate)

```sql
FUNCTION_NAME()
OVER (
    PARTITION BY column   -- optional
    ORDER BY column       -- required
)
```

Important Rules:

- Ranking functions **must not accept arguments** (except NTILE(n)).
- ORDER BY is mandatory.
- PARTITION BY is optional.
- Frame clause is NOT allowed.

---

# 5. ROW_NUMBER()

Assigns a unique sequential integer to each row.

## Example

```sql
SELECT
    OrderID,
    Sales,
    ROW_NUMBER() OVER (ORDER BY Sales DESC) AS RowNum
FROM SalesOrders;
```

### Output (Sorted by Sales DESC)

| Sales | RowNum |
| ----- | ------ |
| 90    | 1      |
| 90    | 2      |
| 60    | 3      |
| 60    | 4      |
| 60    | 5      |
| 30    | 6      |
| 20    | 7      |
| 20    | 8      |
| 20    | 9      |
| 10    | 10     |

Explanation (one line): ROW_NUMBER assigns unique ranks ignoring ties and without gaps.

Execution Flow:

1. Sort by Sales DESC
2. Assign sequential integers

---

# 6. RANK()

Handles ties but leaves gaps.

```sql
SELECT
    Sales,
    RANK() OVER (ORDER BY Sales DESC) AS RankValue
FROM SalesOrders;
```

### Output

| Sales | RankValue |
| ----- | --------- |
| 90    | 1         |
| 90    | 1         |
| 60    | 3         |
| 60    | 3         |
| 60    | 3         |
| 30    | 6         |
| 20    | 7         |
| 20    | 7         |
| 20    | 7         |
| 10    | 10        |

Explanation (one line): RANK shares rank for ties but skips numbers after ties.

---

# 7. DENSE_RANK()

Handles ties without leaving gaps.

```sql
SELECT
    Sales,
    DENSE_RANK() OVER (ORDER BY Sales DESC) AS DenseRankValue
FROM SalesOrders;
```

### Output

| Sales | DenseRankValue |
| ----- | -------------- |
| 90    | 1              |
| 90    | 1              |
| 60    | 2              |
| 60    | 2              |
| 60    | 2              |
| 30    | 3              |
| 20    | 4              |
| 20    | 4              |
| 20    | 4              |
| 10    | 5              |

Explanation (one line): DENSE_RANK shares ranks for ties but keeps ranking sequence continuous.

---

# 8. NTILE(n)

Divides rows into n nearly equal buckets.

```sql
SELECT
    OrderID,
    Sales,
    NTILE(3) OVER (ORDER BY Sales DESC) AS Buckets
FROM SalesOrders;
```

### Example Output (10 rows, 3 buckets)

| Sales | Buckets |
| ----- | ------- |
| 90    | 1       |
| 90    | 1       |
| 60    | 1       |
| 60    | 1       |
| 60    | 2       |
| 30    | 2       |
| 20    | 2       |
| 20    | 3       |
| 20    | 3       |
| 10    | 3       |

Explanation (one line): NTILE divides sorted rows into approximately equal groups where larger groups come first.

Bucket Formula:

bucket_size = total_rows / number_of_buckets

---

# 9. CUME_DIST()

Formula:

CUME_DIST = last_position_of_value / total_rows

```sql
SELECT
    Sales,
    CUME_DIST() OVER (ORDER BY Sales DESC) AS CumDist
FROM SalesOrders;
```

Explanation (one line): CUME_DIST calculates cumulative distribution including current row.

---

# 10. PERCENT_RANK()

Formula:

(position - 1) / (total_rows - 1)

```sql
SELECT
    Sales,
    PERCENT_RANK() OVER (ORDER BY Sales DESC) AS PercentRank
FROM SalesOrders;
```

Explanation (one line): PERCENT_RANK calculates relative rank excluding current row from denominator.

# 11. Real-World Use Cases

## 11.1 Top-N Analysis

Find highest sale per product:

```sql
SELECT *
FROM (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY ProductID ORDER BY Sales DESC) rn
    FROM SalesOrders
) t
WHERE rn = 1;
```

## 11.2 Bottom-N Analysis

Lowest 2 customers by total sales using GROUP BY + ROW_NUMBER.

## 11.3 Remove Duplicates

```sql
SELECT *
FROM (
    SELECT *,
           ROW_NUMBER() OVER (PARTITION BY OrderID ORDER BY CreationTime DESC) rn
    FROM SalesOrders
) t
WHERE rn = 1;
```

## 11.4 Segmentation (NTILE)

Divide customers into High/Medium/Low segments.

## 11.5 Distribution Analysis

Use CUME_DIST to find top 40% products.

# 12. Performance Notes

- ORDER BY requires sorting (expensive operation).
- Index on ORDER BY columns improves performance.
- Partitioning increases memory usage.
- NTILE with large datasets may require full sort.

---

# 13. Common Mistakes

1. Forgetting ORDER BY.
2. Trying to use frame clause.
3. Confusing RANK vs DENSE_RANK.
4. Using ROW_NUMBER when tie-handling required.
5. Filtering without subquery.

---

# 14. Best Practices

- Always define clear ORDER BY.
- Use ROW_NUMBER for deduplication.
- Use RANK for competition-style ranking.
- Use DENSE_RANK for grouped ranking without gaps.
- Use NTILE for segmentation or load balancing.
- Use CUME_DIST for distribution analysis.
- Use PERCENT_RANK for relative positioning.

# Final Summary

Window ranking functions:

- Always sort first
- Assign rank within window
- Support integer or percentage ranking
- Are essential for analytics, segmentation, data quality, and ETL optimization

Mastering ranking functions enables advanced SQL analytics and real-world business insights.
