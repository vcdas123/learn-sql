# Aggregate Functions in SQL

---

# 1. Introduction

Aggregate functions are fundamental tools in SQL for data analysis.

They:

- Accept multiple rows as input
- Return a single summarized value
- Help uncover insights from raw data

Aggregate functions are essential for:

- Business reporting
- KPI tracking
- Financial summaries
- Customer analytics

This chapter covers:

1. Core aggregate functions
2. Execution behavior
3. GROUP BY usage
4. Query outputs
5. Performance considerations
6. Common mistakes
7. Real-world analytics use cases
8. Best practices

---

# 2. What Are Aggregate Functions?

An aggregate function processes multiple rows and produces one summarized result.

Example:

If we have sales values:

10, 15, 20, 35

An aggregate function like `SUM()` returns:

80

---

# 3. Sample Schema Used in This Chapter

```sql
CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    CustomerID INT,
    Sales INT
);
```

```sql
INSERT INTO Orders VALUES

(2, 1, 20),
(3, 2, 15),
(4, 3, 35);
```

---

# 4. COUNT() — Counting Rows

## 4.1 Purpose

Counts rows in a table.

## Syntax

```sql
SELECT COUNT(*) FROM Orders;
```

## Output

| COUNT(\*) |
| --------- |
| 4         |

Execution Flow:

- SQL scans table
- Counts number of rows
- Returns single value

---

# 5. SUM() — Total Value

## Purpose

Adds numeric values together.

## Syntax

```sql
SELECT SUM(Sales) AS TotalSales
FROM Orders;
```

## Calculation

10 + 20 + 15 + 35 = 80

## Output

| TotalSales |
| ---------- |
| 80         |

---

# 6. AVG() — Average Value

## Syntax

```sql
SELECT AVG(Sales) AS AverageSales
FROM Orders;
```

## Calculation

80 / 4 = 20

## Output

| AverageSales |
| ------------ |
| 20           |

Execution Flow:

1. SUM all values
2. Divide by count of non-null rows

---

# 7. MAX() — Highest Value

```sql
SELECT MAX(Sales) AS HighestSales
FROM Orders;
```

## Output

| HighestSales |
| ------------ |
| 35           |

SQL scans values and returns largest.

---

# 8. MIN() — Lowest Value

```sql
SELECT MIN(Sales) AS LowestSales
FROM Orders;
```

## Output

| LowestSales |
| ----------- |
| 10          |

---

# 9. Combining Multiple Aggregates

```sql
SELECT
    COUNT(*) AS TotalOrders,
    SUM(Sales) AS TotalSales,
    AVG(Sales) AS AverageSales,
    MAX(Sales) AS HighestSales,
    MIN(Sales) AS LowestSales
FROM Orders;
```

## Output

| TotalOrders | TotalSales | AverageSales | HighestSales | LowestSales |
| ----------- | ---------- | ------------ | ------------ | ----------- |
| 4           | 80         | 20           | 35           | 10          |

---

# 10. GROUP BY — Breaking Down Aggregation

Without GROUP BY → one global result.

With GROUP BY → aggregation per category.

## Example — Per Customer

```sql
SELECT
    CustomerID,
    COUNT(*) AS TotalOrders,
    SUM(Sales) AS TotalSales,
    AVG(Sales) AS AverageSales,
    MAX(Sales) AS HighestSale,
    MIN(Sales) AS LowestSale
FROM Orders
GROUP BY CustomerID;
```

## Output

| CustomerID | TotalOrders | TotalSales | AverageSales | HighestSale | LowestSale |
| ---------- | ----------- | ---------- | ------------ | ----------- | ---------- |
| 1          | 2           | 30         | 15           | 20          | 10         |
| 2          | 1           | 15         | 15           | 15          | 15         |
| 3          | 1           | 35         | 35           | 35          | 35         |

Execution Flow:

1. SQL groups rows by CustomerID
2. Applies aggregate functions per group
3. Returns one row per group

---

# 11. NULL Behavior in Aggregates

Aggregate functions ignore NULL values (except COUNT(\*)).

Example:

If Sales = 10, 20, NULL

AVG = (10 + 20) / 2 = 15

Important:

```sql
COUNT(Sales)
```

Counts non-null values only.

```sql
COUNT(*)
```

Counts all rows.

---

# 12. Performance Notes

- Aggregates require full table or index scan
- Indexes can speed up MIN() and MAX()
- GROUP BY may require sorting or hashing
- Large aggregations benefit from indexing grouped columns

---

# 13. Common Mistakes

1. Using non-grouped columns without GROUP BY
2. Confusing COUNT(\*) with COUNT(column)
3. Ignoring NULL behavior
4. Forgetting GROUP BY when mixing aggregates and normal columns
5. Assuming AVG includes NULL values

# 14. Real-World Analytics Use Cases

- Sales performance per region
- KPI dashboards
- Financial summaries
- Inventory reporting

---

# 15. Best Practices

- Always alias aggregate columns
- Use COUNT(\*) when counting rows
- Use COUNT(column) when counting non-null values
- Combine aggregates in one query for efficiency
- Index columns used in GROUP BY
- Validate NULL handling
