# Window Aggregate Functions in SQL (COUNT(\*), COUNT(1), COUNT(column), SUM, AVG, MIN, MAX)

---

# 1. Introduction

Window aggregate functions allow you to aggregate data **without losing row-level detail**.

Unlike GROUP BY:

- GROUP BY → collapses rows
- Window aggregates → preserve rows and add aggregated insight

The five core window aggregate functions:

- COUNT()
- SUM()
- AVG()
- MIN()
- MAX()

They aggregate multiple rows within a window and return **one aggregated value per row**.

# 3. Window Aggregate Syntax

```sql
AGG_FUNCTION(expression)
OVER (
    PARTITION BY column
    ORDER BY column
    frame_clause
)
```

Rules from transcript:

- Expression required
- COUNT → accepts any data type
- SUM, AVG, MIN, MAX → require numeric (except MIN/MAX allow comparable types)
- PARTITION BY optional
- ORDER BY optional
- Frame optional

# 4. COUNT() Window Function

## 4.1 Definition

Returns number of rows within each window.

---

## 4.2 COUNT(\*) – Counts All Rows (Includes NULL)

```sql
SELECT
    OrderID,
    COUNT(*) OVER () AS TotalOrders
FROM Orders;
```

### Output

| OrderID | TotalOrders |
| ------- | ----------- |
| 1       | 6           |
| 2       | 6           |
| 3       | 6           |
| 4       | 6           |
| 5       | 6           |
| 6       | 6           |

Explanation (one line): Entire table treated as one window, so total row count (6) is repeated for each row.

---

## 4.3 COUNT(1) – Counts All Rows (Same as COUNT(\*))

```sql
SELECT
    OrderID,
    COUNT(1) OVER () AS TotalOrders_UsingOne
FROM Orders;
```

### Output

| OrderID | TotalOrders_UsingOne |
| ------- | -------------------- |
| 1       | 6                    |
| 2       | 6                    |
| 3       | 6                    |
| 4       | 6                    |
| 5       | 6                    |
| 6       | 6                    |

Explanation (one line): COUNT(1) counts all rows just like COUNT(\*), including NULL rows, because the constant 1 is never NULL.

Important Notes (Transcript Accurate):

- COUNT(\*) and COUNT(1) return identical results.
- NULL values are counted when using \* or 1.
- Performance difference is negligible in most systems.
- Choice between \* and 1 is mostly stylistic.

---

## 4.4 COUNT(column) – Ignores NULL

If Sales contained NULL values:

```sql
COUNT(Sales) OVER ()
```

Explanation: NULL values are ignored when a column is specified.

---

## 4.4 COUNT Per Product

```sql
SELECT
    OrderID,
    ProductID,
    COUNT(*) OVER (PARTITION BY ProductID) AS OrdersPerProduct
FROM Orders;
```

### Output

| OrderID | ProductID | OrdersPerProduct |
| ------- | --------- | ---------------- |
| 1       | 101       | 3                |
| 2       | 101       | 3                |
| 3       | 101       | 3                |
| 4       | 102       | 2                |
| 5       | 102       | 2                |
| 6       | 103       | 1                |

Explanation (one line): Data is partitioned by ProductID and rows are counted within each partition.

# 5. SUM() Window Function

## 5.1 Definition

Returns total sum of numeric values within window.

---

## 5.2 Overall Total Sales

```sql
SELECT
    OrderID,
    Sales,
    SUM(Sales) OVER () AS TotalSales
FROM Orders;
```

### Output

| OrderID | Sales | TotalSales |
| ------- | ----- | ---------- |
| 1       | 10    | 225        |
| 2       | 30    | 225        |
| 3       | 20    | 225        |
| 4       | 15    | 225        |
| 5       | 60    | 225        |
| 6       | 90    | 225        |

Explanation (one line): SUM aggregates entire dataset once (10+30+20+15+60+90=225) and repeats result for each row.

---

## 5.3 SUM Per Product

```sql
SUM(Sales) OVER (PARTITION BY ProductID)
```

### Output

| ProductID | SalesByProduct |
| --------- | -------------- |
| 101       | 60             |
| 102       | 75             |
| 103       | 90             |

Explanation (one line): SUM calculated separately for each product partition.

---

# 6. AVG() Window Function

## 6.1 Definition

Returns average of numeric values within window.

---

## 6.2 Average Per Product

```sql
SELECT
    ProductID,
    Sales,
    AVG(Sales) OVER (PARTITION BY ProductID) AS AvgSales
FROM Orders;
```

### Output

| ProductID | Sales | AvgSales |
| --------- | ----- | -------- |
| 101       | 10    | 20       |
| 101       | 30    | 20       |
| 101       | 20    | 20       |
| 102       | 15    | 37.5     |
| 102       | 60    | 37.5     |
| 103       | 90    | 90       |

Explanation (one line): AVG sums values within partition and divides by row count in that partition.

---

# 7. MIN() and MAX()

## 7.1 Definition

- MIN → lowest value in window
- MAX → highest value in window

---

## 7.2 Highest & Lowest Per Product

```sql
SELECT
    ProductID,
    Sales,
    MIN(Sales) OVER (PARTITION BY ProductID) AS MinSales,
    MAX(Sales) OVER (PARTITION BY ProductID) AS MaxSales
FROM Orders;
```

### Output

| ProductID | Sales | MinSales | MaxSales |
| --------- | ----- | -------- | -------- |
| 101       | 10    | 10       | 30       |
| 101       | 30    | 10       | 30       |
| 101       | 20    | 10       | 30       |
| 102       | 15    | 15       | 60       |
| 102       | 60    | 15       | 60       |
| 103       | 90    | 90       | 90       |

Explanation (one line): MIN and MAX scan each partition to return lowest and highest values respectively.

# 8. Running Total (Very Important Use Case)

```sql
SELECT
    OrderDate,
    Sales,
    SUM(Sales) OVER (
        ORDER BY OrderDate
    ) AS RunningTotal
FROM Orders;
```

### Output

| OrderDate  | Sales | RunningTotal |
| ---------- | ----- | ------------ |
| 2024-01-01 | 10    | 10           |
| 2024-01-02 | 30    | 40           |
| 2024-01-03 | 20    | 60           |
| 2024-01-04 | 15    | 75           |
| 2024-01-05 | 60    | 135          |
| 2024-01-06 | 90    | 225          |

Explanation (one line): ORDER BY triggers default frame (UNBOUNDED PRECEDING to CURRENT ROW) producing cumulative running total.

Execution Flow:

1. Sort by OrderDate
2. Apply default frame
3. Aggregate from first row to current row

---

# 9. Rolling Total (Fixed Window)

Example: 2 previous rows + current

```sql
SUM(Sales) OVER (
    ORDER BY OrderDate
    ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
)
```

Explanation: Window size fixed to 3 rows and shifts forward (rolling effect).

---

# 10. Comparison Analysis (Part-to-Whole)

Find percentage contribution:

```sql
SELECT
    OrderID,
    Sales,
    ROUND(
        CAST(Sales AS FLOAT) /
        SUM(Sales) OVER () * 100,
        2
    ) AS PercentageOfTotal
FROM Orders;
```

Explanation (one line): Each row’s sales divided by total sales to measure contribution percentage.

---

# 11. Filtering Using Window Functions (Subquery Required)

Find orders above average:

```sql
SELECT *
FROM (
    SELECT
        OrderID,
        Sales,
        AVG(Sales) OVER () AS AvgSales
    FROM Orders
) t
WHERE Sales > AvgSales;
```

Explanation (one line): Window functions cannot be used in WHERE directly, so filtering must occur in outer query.

# 12. Performance Notes

- ORDER BY requires sorting
- Large partitions increase memory usage
- Index on partition/order columns improves speed
- Avoid unnecessary window definitions

---

# 13. Common Mistakes

1. Forgetting NULL behavior in COUNT
2. Using window function inside WHERE
3. Forgetting ORDER BY in running total
4. Misunderstanding default frame
5. Mixing GROUP BY incorrectly

# Final Summary

Window aggregate functions:

- Aggregate data per window
- Preserve row-level detail
- Enable advanced analytics

By changing the window definition (PARTITION BY, ORDER BY, frame), you unlock different analytical use cases such as:

- Overall totals
- Group comparisons
- Running totals
- Rolling totals
- Moving averages
- Contribution analysis

Mastering window aggregate functions is essential for professional SQL data analytics.
