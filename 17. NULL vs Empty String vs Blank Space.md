# NULL vs Empty String vs Blank Space

---

# 1. Introduction

One of the most confusing topics in database development is the difference between:

- NULL
- Empty String ('')
- Blank Space (' ')

Although they may look similar in query results, they behave very differently in SQL.

Failing to understand these differences can lead to:

- Data quality issues
- Incorrect analytics
- Storage waste
- Poor performance

This chapter explains the differences clearly and shows how to standardize data using SQL policies.

---

# 2. Conceptual Differences

## 2.1 NULL

Meaning:

- Unknown value
- Missing value
- Not provided

Important:

- Not equal to 0
- Not equal to ''
- Not equal to ' '
- Has no data type

NULL is a special SQL marker.

---

## 2.2 Empty String ('')

Meaning:

- Known value
- But contains zero characters

It is:

- A string
- Length = 0
- Takes storage space

Example:

```
''
```

---

## 2.3 Blank Space (' ')

Meaning:

- Known value
- Contains space character(s)

It is:

- A string
- Length ≥ 1
- Each space consumes storage

Example:

```
' '
```

or

```
'   '
```

---

# 3. Demonstration Using Sample Data

## Sample CTE Data

```sql
WITH Orders AS (
    SELECT 'A' AS Category
    UNION ALL
    SELECT NULL
    UNION ALL
    SELECT ''
    UNION ALL
    SELECT ' '
)
SELECT * FROM Orders;
```

---

# 4. Why They Look the Same

When selecting results:

```sql
SELECT Category FROM Orders;
```

Output may visually appear similar for:

- NULL
- ''
- ' '

This creates confusion.

---

# 5. Detecting Differences Using Length

In SQL Server:

```sql
SELECT
    Category,
    DATALENGTH(Category) AS CategoryLength
FROM Orders;
```

### Output Explanation

| Category | CategoryLength |
| -------- | -------------- |
| A        | 1              |
| NULL     | NULL           |
| ''       | 0              |
| ' '      | 1              |

Key Observations:

- NULL → length is NULL
- Empty string → length = 0
- Blank space → length ≥ 1

Never rely on visual inspection. Always measure length.

---

# 6. Side-by-Side Comparison

| Feature     | NULL    | Empty String    | Blank Space           |
| ----------- | ------- | --------------- | --------------------- |
| Meaning     | Unknown | Known but empty | Known, contains space |
| Data Type   | None    | String          | String                |
| Length      | NULL    | 0               | ≥1                    |
| Storage     | Minimal | Uses storage    | Uses more storage     |
| Performance | Best    | Good            | Worst                 |
| Comparison  | IS NULL | = ''            | = ' '                 |

# 7. Data Policies (Standardization Strategies)

Before analytics, define data policies.

Poor data quality causes inaccurate reports.

We define three common policies.

---

# 8. Policy 1 — Allow NULL and Empty String (Remove Spaces)

Rule:

- Keep NULL
- Keep empty string
- Remove blank spaces

## Implementation Using TRIM()

```sql
SELECT
    TRIM(Category) AS CleanCategory
FROM Orders;
```

Execution Flow:

1. Remove leading spaces
2. Remove trailing spaces
3. Convert blank spaces into empty string

---

# 9. Policy 2 — Use NULL Only (Recommended for Storage)

Rule:

- Convert blank spaces → empty string
- Convert empty string → NULL

## Implementation

```sql
SELECT
    NULLIF(TRIM(Category), '') AS CleanCategory
FROM Orders;
```

Execution Steps:

1. TRIM removes blank spaces
2. NULLIF converts empty string to NULL

Result:

All empty-like values become NULL.

Advantages:

- Saves storage
- Improves performance
- Simplifies analytics

---

# 10. Policy 3 — Use Default Value (e.g., 'Unknown')

Rule:

- Convert blank spaces → NULL
- Convert empty string → NULL
- Convert NULL → 'Unknown'

## Implementation

```sql
SELECT
    COALESCE(
        NULLIF(TRIM(Category), ''),
        'Unknown'
    ) AS CleanCategory
FROM Orders;
```

Execution Steps:

1. TRIM removes spaces
2. NULLIF converts empty to NULL
3. COALESCE replaces NULL with 'Unknown'

Use Case:

Best for reporting layers (Power BI, Tableau).

Not recommended for raw storage (wastes space).

---

# 11. Performance Considerations

Ranking by performance and storage:

1. NULL (best)
2. Empty string
3. Blank spaces (worst)

Reasons:

- NULL does not store characters
- Blank spaces consume storage
- More characters → larger index size → slower queries

---

# 12. Comparison Rules

To search for NULL:

```sql
WHERE Category IS NULL
```

To search for empty string:

```sql
WHERE Category = ''
```

To search for blank space:

```sql
WHERE Category = ' '
```

---

# 13. Why This Matters in Analytics

If not standardized:

- NULL grouped separately
- '' grouped separately
- ' ' grouped separately

Leads to:

- Duplicate categories
- Incorrect counts
- Misleading dashboards

Always clean data before aggregation.

---

# 14. Recommended Strategy

During ETL (Data Loading):

→ Use Policy 2 (Standardize to NULL)

During Reporting Layer:

→ Use Policy 3 (Display 'Unknown')

Avoid Policy 1 in production — it creates confusion.

---

# 15. Final Summary

NULL:

- Unknown
- No value
- Best for storage

Empty String:

- Known
- Zero characters
- Uses storage

Blank Space:

- Known
- One or more spaces
- Wastes storage

Always define clear data policies.

Data cleaning is mandatory before analytics.

Understanding these differences ensures:

- Clean reporting
- Accurate decisions
- Optimized performance
