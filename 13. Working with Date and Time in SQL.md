# 2. Core Concepts: Date, Time, Timestamp

## 2.1 What is a DATE?

A DATE represents a calendar date.

Format (ISO standard):

```
YYYY-MM-DD
```

Example:

```
2025-08-20
```

Components:

- Year (4 digits)
- Month (1–12)
- Day (1–31)

Used for:

- Birth dates
- Order dates
- Deadlines
- Subscription start dates

---

## 2.2 What is TIME?

A TIME represents a specific time of day.

Format:

```
HH:MM:SS
```

Example:

```
18:55:45
```

Components:

- Hour (0–23)
- Minute (0–59)
- Second (0–59)

---

## 2.3 What is TIMESTAMP (or DATETIME)?

A TIMESTAMP combines DATE + TIME.

Example:

```
2025-08-20 18:55:45
```

Hierarchy (from highest to lowest precision):

Year → Month → Day → Hour → Minute → Second → Millisecond

Different databases use different names:

- SQL Server: DATETIME / DATETIME2
- MySQL: DATETIME / TIMESTAMP
- PostgreSQL: TIMESTAMP
- Oracle: TIMESTAMP

# 4. Sources of Date Values in SQL

There are 3 primary ways to work with dates:

## 4.1 Stored Column Values

```sql
SELECT OrderDate FROM SalesOrders;
```

---

## 4.2 Hardcoded Date Literal

```sql
SELECT '2025-08-20' AS HardcodedDate;
```

---

## 4.3 Current System Date

```sql
SELECT GETDATE() AS CurrentDateTime;
```

Execution Flow:

- SQL evaluates GETDATE() once per query execution
- Same value repeats for all rows

# 6. Basic Part Extraction

## 6.1 DAY(), MONTH(), YEAR()

### Syntax

```sql
DAY(date)
MONTH(date)
YEAR(date)
```

### Example

```sql
SELECT
    YEAR(CreationTime) AS OrderYear,
    MONTH(CreationTime) AS OrderMonth,
    DAY(CreationTime) AS OrderDay
FROM SalesOrders;
```

### Output

| OrderYear | OrderMonth | OrderDay |
| --------- | ---------- | -------- |
| 2025      | 1          | 5        |
| 2025      | 1          | 10       |
| 2025      | 2          | 2        |

Return Type: INTEGER

---

# 7. DATEPART()

Extracts numeric parts beyond just year/month/day.

### Syntax

```sql
DATEPART(part, date)
```

### Supported Parts (Common)

- year
- month
- day
- hour
- minute
- second
- quarter
- week
- weekday

### Example

```sql
SELECT
    OrderID,
    DATEPART(quarter, CreationTime) AS Quarter,
    DATEPART(week, CreationTime) AS WeekNumber,
    DATEPART(hour, CreationTime) AS HourPart
FROM SalesOrders;
```

### Example Output

| OrderID | Quarter | WeekNumber | HourPart |
| ------- | ------- | ---------- | -------- |
| 1       | 1       | 1          | 12       |
| 2       | 1       | 2          | 14       |
| 3       | 1       | 5          | 9        |
| 4       | 1       | 7          | 18       |
| 5       | 1       | 9          | 11       |

Return Type: INTEGER

Execution Notes:

- `quarter`, `week`, and `hour` return numeric values.
- Week numbers depend on database regional and DATEFIRST settings.
- Output values are derived from the sample data inserted earlier.

---

# 8. DATENAME()

Returns textual name instead of number.

### Syntax

```sql
DATENAME(part, date)
```

### Available Part Options

The following parts can be used inside `DATENAME()` (and also `DATEPART()` and `DATETRUNC()` in most SQL Server environments):

- year
- quarter
- month
- dayofyear
- day
- week
- weekday
- hour
- minute
- second
- millisecond

Abbreviations are also supported (yy, qq, mm, dd, wk, dw, hh, mi, ss, ms), but using full names is recommended for readability and portability.

---

### Example

```sql
SELECT
    OrderID,
    DATENAME(month, CreationTime) AS MonthName,
    DATENAME(weekday, CreationTime) AS DayName,
    DATENAME(quarter, CreationTime) AS QuarterName
FROM SalesOrders;
```

### Example Output

| OrderID | MonthName | DayName  | QuarterName |
| ------- | --------- | -------- | ----------- |
| 1       | January   | Sunday   | 1           |
| 2       | January   | Friday   | 1           |
| 3       | February  | Sunday   | 1           |
| 4       | February  | Saturday | 1           |
| 5       | March     | Saturday | 1           |

Return Type: VARCHAR

Execution Notes:

- `month` and `weekday` return full textual names.
- `quarter` returns numeric text ("1", "2", etc.) but still as a string.
- Output depends on database language settings.

Use Case: Reporting, dashboards, and user-friendly display formatting.

# 9. DATETRUNC()

Reduces precision by resetting lower hierarchy components.

### Syntax

```sql
DATETRUNC(part, date)
```

### Example: Truncate to Month

```sql
SELECT DATETRUNC(month, CreationTime) AS MonthStart
FROM SalesOrders;
```

### Execution Flow

Original Value:

```
2025-02-15 18:45:50
```

---

After DATETRUNC(hour):

```sql
SELECT DATETRUNC(hour, '2025-02-15 18:45:50');
```

Result:

```
2025-02-15 18:00:00
```

Minutes and seconds are reset.

---

After DATETRUNC(day):

```sql
SELECT DATETRUNC(day, '2025-02-15 18:45:50');
```

Result:

```
2025-02-15 00:00:00
```

Entire time portion is reset.

---

After DATETRUNC(month):

```sql
SELECT DATETRUNC(month, '2025-02-15 18:45:50');
```

Result:

```
2025-02-01 00:00:00
```

Day is reset to 1. Time is reset.

---

After DATETRUNC(year):

```sql
SELECT DATETRUNC(year, '2025-02-15 18:45:50');
```

Result:

```
2025-01-01 00:00:00
```

All lower components (month, day, time) are reset to their minimum valid values.

---

Summary of Reset Behavior:

| Truncation Level | Result Example      |
| ---------------- | ------------------- |
| hour             | 2025-02-15 18:00:00 |
| day              | 2025-02-15 00:00:00 |
| month            | 2025-02-01 00:00:00 |
| year             | 2025-01-01 00:00:00 |

Lower hierarchy components are reset depending on the specified level.

# 10. Aggregation Using DATETRUNC()

## Count Orders per Month

```sql
SELECT
    DATETRUNC(month, CreationTime) AS OrderMonth,
    COUNT(*) AS TotalOrders
FROM SalesOrders
GROUP BY DATETRUNC(month, CreationTime);
```

### Output

| OrderMonth | TotalOrders |
| ---------- | ----------- |
| 2025-01-01 | 2           |
| 2025-02-01 | 2           |
| 2025-03-01 | 1           |

Performance Note: Using functions in GROUP BY may prevent index usage.

Better Approach (if indexed): Store computed month column or use range filtering.

# 11. EOMONTH()

Returns last day of month.

### Syntax

```sql
EOMONTH(date)
```

### Example

```sql
SELECT
    CreationTime,
    EOMONTH(CreationTime) AS EndOfMonth
FROM SalesOrders;
```

---

# 12. Filtering by Date Parts

## Orders in February

```sql
SELECT *
FROM SalesOrders
WHERE MONTH(OrderDate) = 2;
```

⚠ Performance Warning: Using MONTH(OrderDate) prevents index seek.

Better:

```sql
WHERE OrderDate >= '2025-02-01'
AND OrderDate < '2025-03-01'
```

This preserves index efficiency.

---

# 13. DATEADD()

Adds time interval.

```sql
SELECT DATEADD(day, 7, OrderDate) AS NewDate
FROM SalesOrders;
```

---

# 14. DATEDIFF()

Returns difference between two dates.

```sql
SELECT
    DATEDIFF(day, OrderDate, ShipDate) AS ShippingDelay
FROM SalesOrders;
```

---

# 15. ISDATE()

Validates if string is valid date.

```sql
SELECT ISDATE('2025-08-20');
```

Returns: 1 (valid) or 0 (invalid)

---

# 16. Common Mistakes

1. Using string format 'MM-DD-YYYY'
2. Filtering using MONTH() in WHERE clause
3. Ignoring time portion in datetime comparison
4. Assuming week numbers are universal (depends on regional settings)
5. Mixing time zones without conversion

# 18. Best Practices

- Use ISO format: YYYY-MM-DD
- Prefer range filtering over DATEPART in WHERE
- Store timestamps in UTC
- Use DATETRUNC for aggregation
- Avoid formatting inside heavy queries
- Use indexes on date columns
