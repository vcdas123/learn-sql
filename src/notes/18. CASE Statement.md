## 3.1 Full Syntax (Search CASE)

```sql
CASE
    WHEN condition1 THEN result1
    WHEN condition2 THEN result2
    ELSE default_result
END
```

### Key Components

| Keyword | Purpose                    |
| ------- | -------------------------- |
| CASE    | Start conditional logic    |
| WHEN    | Define condition           |
| THEN    | Value if condition is true |
| ELSE    | Default value (optional)   |
| END     | Close logic block          |

---

## 3.2 Execution Flow

SQL evaluates conditions **top to bottom**.

1. Evaluate first condition
2. If TRUE → return value and STOP
3. If FALSE → move to next condition
4. If no condition matches → return ELSE (or NULL if missing)

# 5. Basic Example — Single Condition

```sql
SELECT
    OrderID,
    Sales,
    CASE
        WHEN Sales > 50 THEN 'High'
    END AS Category
FROM Orders;
```

### Output

| OrderID | Sales | Category |
| ------- | ----- | -------- |
| 1       | 60    | High     |
| 2       | 30    | NULL     |
| 3       | 15    | NULL     |
| 4       | 10    | NULL     |
| 5       | 50    | NULL     |
| 6       | 70    | High     |
| 7       | 25    | NULL     |
| 8       | NULL  | NULL     |

Explanation:

- Only Sales > 50 return 'High'
- No ELSE defined → unmatched rows return NULL

# 6. Multiple Conditions

```sql
SELECT
    OrderID,
    Sales,
    CASE
        WHEN Sales > 50 THEN 'High'
        WHEN Sales > 20 THEN 'Medium'
    END AS Category
FROM Orders;
```

### Output

| OrderID | Sales | Category |
| ------- | ----- | -------- |
| 1       | 60    | High     |
| 2       | 30    | Medium   |
| 3       | 15    | NULL     |
| 4       | 10    | NULL     |
| 5       | 50    | Medium   |
| 6       | 70    | High     |
| 7       | 25    | Medium   |
| 8       | NULL  | NULL     |

Execution:

- Order 1: First condition true → stop
- Order 2: First false, second true → Medium
- Order 3: Both false → NULL

# 7. Using ELSE to Avoid NULL

```sql
SELECT
    OrderID,
    Sales,
    CASE
        WHEN Sales > 50 THEN 'High'
        WHEN Sales > 20 THEN 'Medium'
        ELSE 'Low'
    END AS Category
FROM Orders;
```

### Output

| OrderID | Sales | Category |
| ------- | ----- | -------- |
| 1       | 60    | High     |
| 2       | 30    | Medium   |
| 3       | 15    | Low      |
| 4       | 10    | Low      |
| 5       | 50    | Medium   |
| 6       | 70    | High     |
| 7       | 25    | Medium   |
| 8       | NULL  | Low      |

With ELSE defined, no NULL is returned.

# 8. Use Case 1 — Categorizing Data with Aggregation

### Task

Generate total sales per category.

```sql
SELECT
    Category,
    SUM(Sales) AS TotalSales
FROM (
    SELECT
        Sales,
        CASE
            WHEN Sales > 50 THEN 'High'
            WHEN Sales > 20 THEN 'Medium'
            ELSE 'Low'
        END AS Category
    FROM Orders
) AS Categorized
GROUP BY Category
ORDER BY TotalSales DESC;
```

### Output

| Category | TotalSales |
| -------- | ---------- |
| High     | 130        |
| Medium   | 105        |
| Low      | 25         |

Execution Flow:

1. Inner query assigns category
2. Outer query aggregates
3. Sorted descending

---

# 9. CASE Data Type Rule

All THEN and ELSE return values must share compatible data types.

Incorrect:

```sql
CASE
    WHEN Sales > 50 THEN 'High'
    ELSE 0
END
```

Error occurs due to mixed string and integer types.

# 10. Use Case 2 — Mapping Values

## Full Form

```sql
CASE
    WHEN Country = 'Germany' THEN 'DE'
    WHEN Country = 'USA' THEN 'US'
    ELSE 'N/A'
END
```

## Quick Form (Simple CASE)

```sql
CASE Country
    WHEN 'Germany' THEN 'DE'
    WHEN 'USA' THEN 'US'
    ELSE 'N/A'
END
```

Limitations:

- Only equality comparisons
- Single column only

# 11. Use Case 3 — Handling NULL Values

```sql
SELECT
    OrderID,
    CASE
        WHEN Sales IS NULL THEN 0
        ELSE Sales
    END AS CleanSales
FROM Orders;
```

# 12. Use Case 4 — Conditional Aggregation

### Task

Count orders per customer where Sales > 30.

```sql
SELECT
    CustomerID,
    SUM(CASE WHEN Sales > 30 THEN 1 ELSE 0 END) AS HighSalesOrders,
    COUNT(*) AS TotalOrders
FROM Orders
GROUP BY CustomerID;
```

### Output

| CustomerID | HighSalesOrders | TotalOrders |
| ---------- | --------------- | ----------- |
| 1          | 1               | 3           |
| 2          | 1               | 2           |
| 3          | 1               | 2           |
| 4          | 0               | 1           |

Execution:

- CASE creates 1/0 flag
- SUM aggregates only qualifying rows

---

# 13. Performance Notes

- CASE is computed row-by-row
- Avoid excessive nested CASE logic
- Using CASE in WHERE may impact index usage
- Conditional aggregation is efficient and avoids subqueries

---

# 14. Common Mistakes

1. Forgetting ELSE (unexpected NULLs)
2. Incorrect condition ordering
3. Mixing data types
4. Overusing nested CASE blocks
5. Using Simple CASE when complex logic needed

---

# 15. Real-World Analytics Use Cases

- Customer segmentation (High / Medium / Low value)
- Risk scoring
- KPI banding
- Flagging anomalies
- Revenue tier reporting
- Conditional performance metrics
- Converting system codes to readable labels

---

# 16. Best Practices

- Always order conditions from most specific to least specific
- Always define ELSE when possible
- Prefer full CASE for flexibility
- Keep logic readable
- Validate distinct values before mapping
- Use conditional aggregation instead of multiple subqueries
