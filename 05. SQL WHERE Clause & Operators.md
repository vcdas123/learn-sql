# SQL WHERE Clause & Operators – Detailed Notes

## 1. Why Do We Need Conditions in SQL?

In real-world databases, tables often contain **thousands or millions of rows**. Fetching all data is rarely useful.

The **`WHERE` clause** allows us to **filter rows** based on conditions so that only relevant data is returned.

```sql
SELECT *
FROM customers
WHERE condition;
```

A condition evaluates to:

- **TRUE** → row is kept
- **FALSE** → row is removed

---

## 2. Categories of SQL Operators

SQL provides different operators to build filtering logic:

1. **Comparison operators**
2. **Logical operators**
3. **Range operator**
4. **Membership operators**
5. **Search operator**

---

## 3. Comparison Operators

Comparison operators compare **two expressions**.

### General Form

```sql
expression operator expression
```

### What Can Be Compared?

- Column ↔ column
- Column ↔ value
- Function result ↔ value
- Expression ↔ value
- Subquery ↔ value (advanced)

---

### Comparison Operator List

| Operator     | Meaning               |
| ------------ | --------------------- |
| `=`          | Equal                 |
| `!=` or `<>` | Not equal             |
| `>`          | Greater than          |
| `>=`         | Greater than or equal |
| `<`          | Less than             |
| `<=`         | Less than or equal    |

---

### `=` Equal Operator

```sql
SELECT *
FROM customers
WHERE country = 'Germany';
```

---

### `!=` / `<>` Not Equal Operator

```sql
SELECT *
FROM customers
WHERE country <> 'Germany';
```

---

### Greater Than / Less Than

```sql
SELECT *
FROM customers
WHERE score > 500;
```

```sql
SELECT *
FROM customers
WHERE score <= 500;
```

---

## 4. Logical Operators

Logical operators combine or modify conditions.

### Logical Operator List

| Operator | Description                         |
| -------- | ----------------------------------- |
| `AND`    | All conditions must be TRUE         |
| `OR`     | At least one condition must be TRUE |
| `NOT`    | Reverses the condition              |

---

### `AND` Operator

```sql
SELECT *
FROM customers
WHERE country = 'USA'
AND score > 500;
```

Both conditions must be true.

---

### `OR` Operator

```sql
SELECT *
FROM customers
WHERE country = 'USA'
OR score > 500;
```

At least one condition must be true.

---

### `NOT` Operator

```sql
SELECT *
FROM customers
WHERE NOT (score < 500);
```

Equivalent to:

```sql
WHERE score >= 500;
```

---

## 5. Range Operator – BETWEEN

Checks whether a value falls **within a range (inclusive)**.

```sql
SELECT *
FROM customers
WHERE score BETWEEN 100 AND 500;
```

### Equivalent Without BETWEEN

```sql
WHERE score >= 100 AND score <= 500;
```

---

## 6. Membership Operators – IN / NOT IN

### `IN`

```sql
SELECT *
FROM customers
WHERE country IN ('Germany', 'USA');
```

### `NOT IN`

```sql
SELECT *
FROM customers
WHERE country NOT IN ('Germany', 'USA');
```

### When to Use IN

- Multiple `OR` conditions on same column
- Cleaner and faster queries
