# SQL Date Formatting and Casting — Advanced Notes

---

# 1. Understanding Date Formats

Before learning formatting and casting functions, it is critical to understand what a **date format** actually means.

A datetime value contains:

```
2025-08-20 18:55:45
```

This contains both:

- Numbers (2025, 08, 20, 18, 55, 45)
- Characters (-, :, space)

A format describes **how this value is represented as text**.

---

## 1.1 Format Specifiers

Each part of the date has a symbolic representation called a **format specifier**.

| Component       | Specifier | Meaning |
| --------------- | --------- | ------- |
| Year (4 digit)  | YYYY      | 2025    |
| Month (2 digit) | MM        | 08      |
| Day (2 digit)   | dd        | 20      |
| Hour (24h)      | HH        | 18      |
| Minute          | mm        | 55      |
| Second          | ss        | 45      |

⚠ Case Sensitive:

- `MM` = Month
- `mm` = Minute

Full ISO Format Representation:

```
yyyy-MM-dd HH:mm:ss
```

---

# 2. Global Date Format Standards

Different regions use different date representations.

| Region             | Format     | Example    |
| ------------------ | ---------- | ---------- |
| ISO (SQL Standard) | YYYY-MM-dd | 2025-08-20 |
| USA                | MM-dd-yyyy | 08-20-2025 |
| Europe             | dd-MM-yyyy | 20-08-2025 |

SQL Server follows ISO standard internally.

Best Practice: Always store dates in ISO format.

---

# 3. What is Formatting?

Formatting changes **how a value looks**, not its data type.

Example:

Original (DATE type):

```
2025-08-20
```

Formatted Output (String):

```
08/20/25
```

The value becomes text.

---

# 4. FORMAT() Function

## 4.1 Syntax

```sql
FORMAT(value, format_string, culture)
```

- value → Date or number
- format_string → Desired output format
- culture → Optional regional culture (e.g., 'en-US', 'fr-FR')

---

## 4.2 Basic Example

```sql
SELECT FORMAT('2025-08-20', 'MM/dd/yyyy') AS USA_Format;
```

### Output

| USA_Format |
| ---------- |
| 08/20/2025 |

---

## 4.3 European Format

```sql
SELECT FORMAT('2025-08-20', 'dd-MM-yyyy') AS Europe_Format;
```

### Output

| Europe_Format |
| ------------- |
| 20-08-2025    |

---

## 4.4 Custom Complex Format Example

Required Format:

```
Day Wed Aug Q1 2025 06:55:45 PM
```

Query:

```sql
SELECT
    'Day ' +
    FORMAT(CreationTime, 'ddd MMM ') +
    'Q' + DATENAME(quarter, CreationTime) +
    ' ' + FORMAT(CreationTime, 'yyyy hh:mm:ss tt')
AS CustomFormat
FROM SalesOrders;
```

### Example Output

| CustomFormat                    |
| ------------------------------- |
| Day Sun Jan Q1 2025 12:10:22 PM |

---

## 4.5 Number Formatting

```sql
SELECT
    FORMAT(1000000, 'N') AS NumericFormat,
    FORMAT(1000, 'C') AS CurrencyFormat,
    FORMAT(0.25, 'P') AS PercentageFormat;
```

### Output

| NumericFormat | CurrencyFormat | PercentageFormat |
| ------------- | -------------- | ---------------- |
| 1,000,000.00  | $1,000.00      | 25.00 %          |

---

## Performance Warning

`FORMAT()` is slower because it uses CLR internally.

Avoid using FORMAT() inside large aggregations.

---

# 5. What is Casting?

Casting changes the **data type**, not just appearance.

Example:

```
'123' (string) → 123 (integer)
```

---

# 6. CAST() Function

## Syntax

```sql
CAST(value AS datatype)
```

---

## 6.1 String to Integer

```sql
SELECT CAST('123' AS INT) AS StringToInt;
```

### Output

| StringToInt |
| ----------- |
| 123         |

---

## 6.2 String to Date

```sql
SELECT CAST('2025-08-20' AS DATE) AS StringToDate;
```

### Output

| StringToDate |
| ------------ |
| 2025-08-20   |

---

## 6.3 Datetime to Date

```sql
SELECT
    CreationTime,
    CAST(CreationTime AS DATE) AS DateOnly
FROM SalesOrders;
```

### Output

| CreationTime        | DateOnly   |
| ------------------- | ---------- |
| 2025-01-05 12:10:22 | 2025-01-05 |

Time portion is removed.

---

# 7. CONVERT() Function

CONVERT can perform both casting and formatting.

## Syntax

```sql
CONVERT(datatype, value, style)
```

- datatype → Target type
- value → Source value
- style → Optional format code

---

## 7.1 Casting Example

```sql
SELECT CONVERT(INT, '123') AS ConvertToInt;
```

---

## 7.2 Formatting Using Style

```sql
SELECT
    CONVERT(VARCHAR, '2025-08-20', 101) AS USA_Format,
    CONVERT(VARCHAR, '2025-08-20', 103) AS Europe_Format;
```

### Output

| USA_Format | Europe_Format |
| ---------- | ------------- |
| 08/20/2025 | 20/08/2025    |

---

## Common Style Codes

| Style | Format              |
| ----- | ------------------- |
| 101   | mm/dd/yyyy          |
| 103   | dd/mm/yyyy          |
| 112   | yyyymmdd            |
| 120   | yyyy-mm-dd hh:mi:ss |

---

# 8. FORMAT vs CONVERT vs CAST

| Feature          | CAST | CONVERT | FORMAT |
| ---------------- | ---- | ------- | ------ |
| Change datatype  | ✅   | ✅      | ❌     |
| Format date      | ❌   | ✅      | ✅     |
| Format numbers   | ❌   | ❌      | ✅     |
| Performance      | Fast | Fast    | Slow   |
| Supports culture | ❌   | ❌      | ✅     |

---

# 9. Real‑World Use Cases

- Standardizing incoming API date formats
- Cleaning CSV imports
- Formatting financial reports
- Building user-friendly dashboards
- Converting strings before inserting into tables

---

# 10. Best Practices

- Store dates in ISO format
- Use CAST for datatype changes
- Use CONVERT for SQL Server style formatting
- Avoid FORMAT in large queries
- Never store formatted dates in database tables

---

# Final Summary

Formatting = Change appearance (returns string)

Casting = Change data type

CONVERT = Casting + Optional formatting

FORMAT = Powerful formatting (slower)

Understanding this distinction is essential for writing clean, performant SQL.
