# Handling NULL Values in SQL

---

## 1. Introduction

NULL handling is one of the most important and misunderstood topics in SQL. Improper handling of NULL values can lead to incorrect analytics, broken joins, inaccurate aggregations, and unexpected results.

This chapter provides:

- Clear conceptual explanation
- SQL syntax examples
- Sample schemas and data
- Query outputs
- Execution flow explanation
- Performance notes
- Real-world analytics use cases
- Common mistakes
- Best practices
- Summary comparison table

---

## 2. What is NULL in SQL?

### 2.1 Definition

A NULL in SQL represents:

- Missing value
- Unknown value
- Not yet provided value

Important:

- NULL ≠ 0
- NULL ≠ empty string ('')
- NULL ≠ blank space (' ')

NULL means "no value exists".

# 5. NULL Handling Functions

---

## 5.1 ISNULL (or IFNULL / NVL depending on DB)

### Purpose

Replace NULL with a specified value.

### Syntax

```sql
ISNULL(expression, replacement_value)
```

### Example

```sql
SELECT customer_id,
       ISNULL(score, 0) AS score_cleaned
FROM customers;
```

### Output

| customer_id | score_cleaned |
| ----------- | ------------- |
| 1           | 800           |
| 2           | 500           |
| 3           | 700           |
| 4           | 900           |
| 5           | 0             |

### Execution Flow

1. Check score
2. If NULL → return 0
3. Otherwise → return score

### Performance Note

- Slightly faster than COALESCE in SQL Server
- Not ANSI standard

---

## 5.2 COALESCE

### Purpose

Return first non-NULL value from a list.

### Syntax

```sql
COALESCE(value1, value2, value3, ...)
```

### Example

```sql
SELECT customer_id,
       COALESCE(score, 0) AS score_cleaned
FROM customers;
```

### Multi-Column Example

```sql
SELECT COALESCE(last_name, 'Unknown') AS last_name_cleaned
FROM customers;
```

### Execution Flow

1. Evaluate from left to right
2. Return first non-NULL
3. Stop evaluation

### Performance Note

- ANSI standard
- Portable across databases

---

## 5.3 NULLIF

### Purpose

Return NULL if two expressions are equal.

### Syntax

```sql
NULLIF(expression1, expression2)
```

### Example

```sql
SELECT order_id,
       NULLIF(quantity, 0) AS quantity_cleaned
FROM orders;
```

If quantity = 0 → return NULL
Otherwise → return quantity

---

## 5.4 IS NULL and IS NOT NULL

### Purpose

Check for NULL values.

### Syntax

```sql
column IS NULL
column IS NOT NULL
```

### Example

```sql
SELECT *
FROM customers
WHERE score IS NULL;
```

---

# 6. NULL and Aggregations

### Key Rule

Aggregate functions ignore NULL values.

### Example

```sql
SELECT AVG(score) FROM customers;
```

NULL scores are ignored.

### If Business Treats NULL as Zero

```sql
SELECT AVG(COALESCE(score,0)) FROM customers;
```

### Real-World Use Case

- Customer average score reporting
- Revenue forecasting
- KPI calculations

---

# 7. NULL in Mathematical Operations

### Rule

Any arithmetic with NULL returns NULL.

```sql
SELECT score + 10 FROM customers;
```

If score is NULL → result NULL

### Safe Version

```sql
SELECT COALESCE(score,0) + 10 AS bonus_score
FROM customers;
```

---

# 8. NULL in Joins

### Problem

NULL values in join keys never match.

### Incorrect Join

```sql
SELECT *
FROM tableA a
JOIN tableB b
  ON a.type = b.type;
```

If type is NULL → no match

### Safe Join Technique

```sql
ON COALESCE(a.type,'') = COALESCE(b.type,'')
```

### Real-World Risk

- Missing transactions
- Incomplete reports
- Incorrect analytics

---

# 9. NULL and Sorting

### Behavior

- ASC → NULLs appear first
- DESC → NULLs appear last

### Force NULLs Last

```sql
ORDER BY
    CASE WHEN score IS NULL THEN 1 ELSE 0 END,
    score ASC;
```

---

# 10. Prevent Division by Zero Using NULLIF

### Problem

```sql
SELECT sales / quantity FROM orders;
```

If quantity = 0 → Error

### Solution

```sql
SELECT sales / NULLIF(quantity, 0) AS price
FROM orders;
```

### Execution Flow

1. If quantity = 0 → convert to NULL
2. Division by NULL → returns NULL
3. Prevents runtime error

---

# 11. Left Anti Join Using IS NULL

### Goal

Find customers with no orders.

```sql
SELECT c.*
FROM customers c
LEFT JOIN orders o
    ON c.customer_id = o.customer_id
WHERE o.customer_id IS NULL;
```

### Use Case

- Customers without purchases
- Users without activity
- Products without sales

---

# 12. Common Mistakes

1. Using = NULL instead of IS NULL
2. Forgetting NULL in joins
3. Ignoring NULL in aggregates
4. Using static large numbers for sorting
5. Assuming NULL equals empty string

---

# 13. Best Practices

- Prefer COALESCE for portability
- Handle NULL before math operations
- Clean NULLs before aggregation
- Never join directly on nullable keys
- Use NULLIF for safe division
- Always validate business meaning of NULL

---

# 14. Performance Considerations

- ISNULL slightly faster in SQL Server
- COALESCE portable but may evaluate multiple expressions
- Avoid wrapping indexed columns in functions (may break index usage)

Understanding how SQL evaluates NULL step-by-step is essential for writing correct and production-ready queries.

# B. Aggregation — Exact Numerical Result

```sql
SELECT AVG(Score) AS AvgScore FROM Customers;
```

Scores used in calculation:
1000, 800, 700, 600 (NULL ignored)

Calculation:
(1000 + 800 + 700 + 600) / 4 = 775

### Output

| AvgScore |
| -------- |
| 775      |

If business treats NULL as 0:

```sql
SELECT AVG(COALESCE(Score,0)) AS AvgScoreIncludingNull
```

(1000 + 800 + 700 + 600 + 0) / 5 = 620

### Output

| AvgScoreIncludingNull |
| --------------------- |
| 620                   |

# C. NULL in JOIN — Full Example

### Sample Tables

Customers:

| CustomerID | Name   |
| ---------- | ------ |
| 1          | Joseph |
| 2          | Sara   |
| 3          | Mary   |

Orders:

| OrderID | CustomerID |
| ------- | ---------- |
| 101     | 1          |
| 102     | NULL       |

```sql
SELECT C.CustomerID, C.Name, O.OrderID
FROM Customers C
LEFT JOIN Orders O
    ON C.CustomerID = O.CustomerID;
```

### Output

| CustomerID | Name   | OrderID |
| ---------- | ------ | ------- |
| 1          | Joseph | 101     |
| 2          | Sara   | NULL    |
| 3          | Mary   | NULL    |

Explanation:

- Order 102 not matched because NULL never equals NULL
