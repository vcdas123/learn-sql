# Advanced SQL Documentation — Handling NULL Values

---

# 1. Introduction

NULL handling is one of the most critical topics in SQL development.

In real-world databases:

- Some fields are required
- Some fields are optional
- Missing data is common

SQL represents missing or unknown values using **NULL**.

Understanding how NULL behaves — especially in calculations, joins, filtering, and sorting — is essential for writing correct and performant SQL queries.

This chapter covers:

1. What NULL means
2. NULL handling functions
3. NULL behavior in aggregation
4. NULL behavior in joins
5. NULL behavior in sorting
6. Advanced use cases

9. Best practices

# 2. What is NULL?

## 2.1 Definition

In SQL, NULL means:

- Unknown value
- Missing value
- Not yet provided

NULL is:

- NOT equal to 0
- NOT equal to empty string ''
- NOT equal to blank space
- NOT equal to anything (even another NULL)

NULL represents absence of value.

# 3. NULL Comparison Rules

Incorrect:

```sql
WHERE Score = NULL
```

Correct:

```sql
WHERE Score IS NULL
```

Why?

Because NULL cannot be compared using =.

---

# 4. ISNULL() Function (SQL Server)

## 4.1 Purpose

Replaces NULL with a specified value.

## Syntax

```sql
ISNULL(expression, replacement_value)
```

---

## Example 1 — Replace with Static Value

```sql
SELECT
    CustomerID,
    ISNULL(Score, 0) AS ScoreWithDefault
FROM Customers;
```

### Output

| CustomerID | ScoreWithDefault |
| ---------- | ---------------- |
| 1          | 1000             |
| 2          | 800              |
| 3          | 700              |
| 4          | 600              |
| 5          | 0                |

---

## Execution Flow

For each row:

IF Score IS NULL → return 0
ELSE → return Score

---

# 5. COALESCE()

## 5.1 Purpose

Returns the first non-NULL value from a list.

## Syntax

```sql
COALESCE(value1, value2, value3, ...)
```

---

## Example

```sql
SELECT
    CustomerID,
    COALESCE(LastName, 'N/A') AS SafeLastName
FROM Customers;
```

---

## Advanced Example (Multiple Columns)

```sql
SELECT
    COALESCE(LastName, FirstName, 'Unknown') AS NamePriority
FROM Customers;
```

Execution Logic:

1. Check LastName
2. If NULL → Check FirstName
3. If NULL → Return 'Unknown'

Stops at first non-null.

---

# 6. ISNULL vs COALESCE

| Feature      | ISNULL          | COALESCE        |
| ------------ | --------------- | --------------- |
| Arguments    | 2 only          | Multiple        |
| Standard SQL | ❌              | ✅              |
| Portability  | Low             | High            |
| Performance  | Slightly faster | Slightly slower |

Recommendation: Prefer COALESCE for portability.

# 7. NULL in Aggregation

## 7.1 Behavior

NULL values are ignored in:

- SUM()
- AVG()
- MIN()
- MAX()

Exception:

```sql
COUNT(*)
```

Counts rows (including NULL rows).

---

## Example

```sql
SELECT AVG(Score) FROM Customers;
```

NULL scores are ignored.

If business treats NULL as 0:

```sql
SELECT AVG(COALESCE(Score, 0)) FROM Customers;
```

---

# 8. NULL in Mathematical Operations

Any arithmetic with NULL returns NULL.

```sql
SELECT 5 + NULL;  -- returns NULL
```

---

## Real Example — Bonus Points

```sql
SELECT
    CustomerID,
    COALESCE(Score, 0) + 10 AS ScoreWithBonus
FROM Customers;
```

Execution Flow:

1. Replace NULL with 0
2. Add 10

---

# 9. NULL in JOIN Conditions

NULL never equals NULL.

```sql
NULL = NULL  -- Unknown
```

This can cause missing join matches.

---

## Sample Schema

```sql
CREATE TABLE Orders (
    OrderID INT,
    CustomerID INT
);
```

---

## Handling NULL in Join

```sql
SELECT *
FROM TableA A
JOIN TableB B
  ON ISNULL(A.KeyCol, '') = ISNULL(B.KeyCol, '');
```

Execution Flow:

1. Replace NULL with empty string
2. Compare safely

---

# 10. NULL in Sorting

Ascending:
NULL appears first.

Descending:
NULL appears last.

---

## Force NULLs Last (Professional Way)

```sql
SELECT *
FROM Customers
ORDER BY
    CASE WHEN Score IS NULL THEN 1 ELSE 0 END,
    Score;
```

---

# 11. NULLIF()

## Purpose

Returns NULL if two values are equal.

## Syntax

```sql
NULLIF(value1, value2)
```

---

## Example — Replace -1 With NULL

```sql
SELECT NULLIF(Price, -1) FROM Products;
```

---

# 12. Prevent Division by Zero

```sql
SELECT
    Sales / NULLIF(Quantity, 0) AS Price
FROM Orders;
```

If Quantity = 0 → becomes NULL → avoids runtime error.

---

# 13. IS NULL / IS NOT NULL

Used for filtering.

```sql
SELECT *
FROM Customers
WHERE Score IS NULL;
```

---

# 14. Anti Join (Advanced)

## Left Anti Join

Find customers without orders.

```sql
SELECT C.*
FROM Customers C
LEFT JOIN Orders O
  ON C.CustomerID = O.CustomerID
WHERE O.CustomerID IS NULL;
```

Returns:
Customers not matching Orders.

# 15. Common Mistakes

1. Using = NULL instead of IS NULL
2. Forgetting NULL handling before math
3. Losing rows in joins due to NULL keys
4. Incorrect aggregation results
5. Using static extreme values for sorting

---

# 16. Performance Notes

- ISNULL slightly faster in SQL Server
- COALESCE portable across DBMS
- Avoid functions on indexed columns in WHERE
- Handle NULL before aggregation

- Always check join keys for NULL

- Avoid hiding data issues silently
