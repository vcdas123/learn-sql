# SQL Query Basics – Detailed Notes

## 1. What is an SQL Query?

An **SQL query** is a way to communicate with a database to ask questions about the data stored inside it. Data is stored in **tables**, tables are stored in **databases**, and SQL (Structured Query Language) is the language used to interact with them.

- When a business asks questions like:
  - _What is the total sales?_
  - _How many customers do we have?_

- We translate those questions into **SQL queries**.

A query:

1. Is written using SQL syntax (mostly `SELECT` statements).
2. Is sent to the database.
3. Is processed by the database engine.
4. Returns a **result set** (rows and columns).

👉 **Important:** `SELECT` queries are **read-only**. They do **not** modify data or database structure.

---

## 2. SQL Query Clauses

An SQL query is made up of different **clauses** (sections). Each clause has a specific role.

Common clauses:

- `SELECT`
- `FROM`
- `WHERE`
- `ORDER BY`
- `GROUP BY`
- `HAVING`
- `DISTINCT`
- `TOP` (or `LIMIT` in some databases)

These clauses allow you to:

- Choose columns
- Filter rows
- Sort data
- Aggregate data
- Remove duplicates
- Limit results

---

## 3. SELECT and FROM (Basic Query)

### Purpose

- `SELECT`: specifies **which columns** you want.
- `FROM`: specifies **which table** the data comes from.

### Example

```sql
SELECT *
FROM customers;
```

### How SQL Executes This

1. `FROM` → fetches all rows from the `customers` table.
2. `SELECT *` → keeps all columns.

Result: all rows and all columns from the table.

### Single-line comment

```sql

```

### Multi-line comment

```sql

This is a
multi-line comment

```

---

## 5. Selecting Specific Columns

Instead of selecting everything, you can choose only the columns you need.

### Example

```sql
SELECT first_name, country, score
FROM customers;
```

### Key Points

- Columns are separated by commas.
- **No comma after the last column**.
- Output column order follows the order in `SELECT`.

This improves performance and makes results cleaner.

---

## 6. WHERE Clause (Filtering Rows)

### Purpose

`WHERE` filters rows **before aggregation** based on a condition.

### Syntax

```sql
SELECT *
FROM customers
WHERE score <> 0;
```

### Example Conditions

- Numeric comparison:

  ```sql
  WHERE score > 500
  ```

- String comparison (must use single quotes):

  ```sql
  WHERE country = 'Germany'
  ```

### Execution Logic

- Each row is checked against the condition.
- Rows that do not meet the condition are removed.

---

## 7. ORDER BY (Sorting Data)

### Purpose

Sorts the result set.

### Syntax

```sql
ORDER BY column_name ASC | DESC
```

- `ASC` (ascending) → default
- `DESC` (descending)

### Example

```sql
SELECT *
FROM customers
ORDER BY score DESC;
```

### Nested Sorting (Multiple Columns)

```sql
ORDER BY country ASC, score DESC;
```

### Important Notes

- Sorting happens **after filtering**.
- Column order in `ORDER BY` defines priority.

---

## 8. GROUP BY (Aggregation)

### Purpose

Groups rows with the same values and applies **aggregate functions**.

Common aggregate functions:

- `SUM()`
- `AVG()`
- `COUNT()`
- `MIN()`
- `MAX()`

### Example

```sql
SELECT country, SUM(score) AS total_score
FROM customers
GROUP BY country;
```

### Key Rules

- Every non-aggregated column in `SELECT` **must appear in `GROUP BY`**.
- `GROUP BY` reduces rows (one row per group).

---

## 9. HAVING (Filtering After Aggregation)

### Purpose

Filters **aggregated results**.

### Difference Between WHERE and HAVING

| WHERE                          | HAVING                   |
| ------------------------------ | ------------------------ |
| Filters rows                   | Filters groups           |
| Before aggregation             | After aggregation        |
| Cannot use aggregate functions | Uses aggregate functions |

### Example

```sql
SELECT country, AVG(score) AS avg_score
FROM customers
GROUP BY country
HAVING AVG(score) > 430;
```

---

## 10. DISTINCT (Removing Duplicates)

### Purpose

Removes duplicate values from the result set.

### Syntax

```sql
SELECT DISTINCT country
FROM customers;
```

### Notes

- Works on selected columns.
- Expensive operation → use only when needed.
- Avoid using on already unique columns (like IDs).

---

## 11. TOP (Limiting Rows)

### Purpose

Limits the number of rows returned.

### Basic Syntax (TOP)

```sql
SELECT TOP 3 *
FROM customers;
```

### Combined with ORDER BY

```sql
SELECT TOP 3 *
FROM customers
ORDER BY score DESC;
```

This is useful for:

- Top-N analysis
- Recent records
- Summary reports

### TOP with OFFSET (Pagination)

In some SQL dialects (like SQL Server using `OFFSET … FETCH`), you may want to **skip a certain number of rows** and then fetch the next set. This is commonly used for **pagination**.

### Syntax (OFFSET + FETCH)

```sql
SELECT *
FROM customers
ORDER BY score DESC
OFFSET 2 ROWS
FETCH NEXT 3 ROWS ONLY;
```

### How It Works

- `OFFSET 2 ROWS` → skips the first 2 rows
- `FETCH NEXT 3 ROWS ONLY` → returns the next 3 rows

So this query:

- Ignores the first 2 highest scores
- Returns the next 3 customers

### Important Notes

- `ORDER BY` is **mandatory** when using `OFFSET` and `FETCH`.
- `TOP` alone limits rows from the beginning of the result set.
- `OFFSET + FETCH` allows precise control over **which rows** are returned.

### Equivalent in Other Databases

```sql
-- MySQL / PostgreSQL
SELECT *
FROM customers
ORDER BY score DESC
LIMIT 3 OFFSET 2;
```

### When to Use

- Pagination in applications (page 2, page 3, etc.)
- Skipping already processed records
- Batch-wise data processing

## 12. Coding Order vs Execution Order

### Coding Order (Must Follow This)

1. `SELECT`
2. `DISTINCT`
3. `TOP`
4. Column list
5. `FROM`
6. `WHERE`
7. `GROUP BY`
8. `HAVING`
9. `ORDER BY`

### Execution Order (How SQL Runs It)

1. `FROM`
2. `WHERE`
3. `GROUP BY`
4. `HAVING`
5. `SELECT`
6. `DISTINCT`
7. `ORDER BY`
8. `TOP`

👉 Understanding execution order helps in writing **correct and optimized queries**.

---

## 13. Types of Filtering in SQL

1. **Column filtering** → `SELECT`
2. **Duplicate filtering** → `DISTINCT`
3. **Row count filtering** → `TOP`
4. **Row condition filtering (before aggregation)** → `WHERE`
5. **Group condition filtering (after aggregation)** → `HAVING`
